第三章 项目开发设计与实现
3.1 开发环境与技术栈选型
3.1.1 硬件环境
开发主机：CPU 8 核及以上，内存≥16GB，磁盘空余≥20GB（用于存储生成文稿、图片资源、打包文件）
演示设备：普通 PC / 笔记本，无独立 GPU 也可运行课堂 Demo 调度框架
3.1.2 软件环境
系统：Windows 10+/macOS/Linux Ubuntu
运行环境：Python 3.9~3.11
版本管理：Git 2.30+、GitHub 远程仓库
依赖包管理：pip /poetry
3.1.3 核心技术栈
表格
模块	技术选型	用途说明
主调度并发框架	threading/multiprocessing	实现四大子任务并行调度、任务阻塞等待、进程状态监听
AI 文本生成	大模型 API 封装（兼容本地大模型 / 在线 API）	选题、正文、参考文献、自媒体文案生成

3.1.4 Demo 安全裁剪技术说明
通过全局配置开关 IS_CLASS_DEMO = True 控制高危功能禁用：
关闭网络爬虫请求类接口、数据库爬取逻辑；
删除第三方平台账号登录、cookie 存储、自动发布接口；
文献数据替换为内置模拟样本库，无需联网抓取。
3.2 系统总体架构设计
整体采用主调度器 + 四大解耦并行子任务分层架构，低耦合、可独立扩展，所有模块仅通过本地文件交换数据，无跨模块内存共享。
3.2.1 分层架构
配置层：全局配置文件，区分正式版 / 课堂 Demo，定义输出目录、字数阈值（5000 字）、参考文献下限（40 篇）、并发参数；
调度核心层：TaskScheduler 主调度类，负责任务创建、并发启动、执行监听、完成校验、资源汇总打包；
业务子任务层：四大独立任务模块，每个模块拥有独立入口函数、文件输出逻辑；
文件输出层：统一本地目录规范，各任务输出独立文件夹，调度器统一汇总打包；
日志输出层：全局统一日志，记录每个任务启动时间、生成文件、异常信息。
